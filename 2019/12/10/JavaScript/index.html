DOM文档对象模型

DOM(Document Object Model)：文档对象模型。

直白的讲就是Java程序解析结构化文档的时候，在内存中生成的包含当前结构化文档所有内容的一个对象模型。所以说DOM并不是一门技术，而是一门思想，或者更明确的讲是一门访问结构化文档的一种方式。

HTML就是一个文档，我们可以通过Java语言去解析它，Java是面向对象的语言，可以把HTML里面的每一个元素解析成一个一个的对象；因为只要封装成对象，在程序里面就可以拿到这些内容;

  • D Document 文档(HTML文档)
  • O Object 对象(Java对象)
  • M Model 模型（读取文档元素，封装成个个对象，这每个对象就相当于该文档的一个模型）

直白的讲，一个实体文件要在内存中形成这么一个对象，这中间肯定需要一些代码的转换操作，而在此我们称呼为DOM解析器。DOM解析器的作用就是完成结构化文档和DOM树之间的转换关系。

事件冒泡或事件捕获？

事件传递有两种方式：冒泡和捕获。

事件传递定义了元素事件触发顺序，如果将<p>元素插入到<div>元素中，用户点击<p>元素，哪一个元素的”click”事件先被触发呢？

在冒泡中，内部元素的事件会先被触发，然后再触发外部元素，即:<p>元素的点击事件先被触发，然后会触发<div>元素的点击事件；

在捕获中，外部元素的事件会先被触发，然后再触发外部元素，即:<div>元素的点击事件先被触发，然后会触发<p>元素的点击事件；

addEventListener()方法可以指定”useCapture”参数来设置传递类型：默认值false即冒泡方式，值为ture时，事件使用捕获传递。
